Quick Start

From bare metal to running VMs — using kctl for the full lifecycle.

kcore is alpha software. This guide assumes you have a dedicated machine or lab environment for testing.

Prerequisites

  • Nix with flakes enabled (for building the ISO)
  • Go 1.22+
  • Make
  • A USB drive (4 GB+)
  • A dedicated x86_64 machine with KVM support (Intel VT-x or AMD-V)

1) Build the ISO

git clone https://github.com/rtacconi/kcore.git
cd kcore
make build-iso

The ISO includes the kcore node-agent, libvirtd, virtlogd, the Automator API, and all dependencies.

2) Write to USB and Boot

USB_DEVICE=/dev/sdX make write-usb

Boot from USB and log in with root / kcore. The network auto-configures via DHCP.

3) Initialize the Cluster

On your workstation, create a cluster identity and CA:

kctl init cluster --name my-cluster

This generates a cluster CA under ~/.kcore/my-cluster/. The CA key stays on your machine.

4) Discover Hardware

Query the node for available disks and NICs:

kctl get disks --node <node-id>
kctl get nics --node <node-id>

5) Install the Node

Trigger OS installation with multi-disk support:

kctl install node --node <node-id> --os-disk /dev/sda --storage-disk /dev/nvme0n1 \
  --hostname kcore01 --run-controller --controller-address 10.0.0.1:9090

Or declaratively:

kctl apply -f node-install.yaml

Example node-install.yaml manifest:

apiVersion: kcore.io/v1
kind: NodeInstall
metadata:
  name: kcore01
spec:
  nodeId: "<node-id>"
  osDisk: /dev/sda
  storageDisk: /dev/nvme0n1
  hostname: kcore01
  runController: true
  controllerAddress: "10.0.0.1:9090"

After reboot, the node auto-registers with the controller via mTLS.

6) Configure Networking

kctl configure network --node <node-id> --bridge br0 --bridge-ports enp1s0 --dns 1.1.1.1 --apply

Or declaratively with kctl apply -f node-network.yaml (kind: NodeNetwork).

7) Configure kctl

kctl config lives at ~/.kcore/config:

current-context: local
contexts:
  local:
    controller: "localhost:9090"
    insecure: true

For TLS:

contexts:
  production:
    controller: "controller.example.com:9090"
    insecure: false
    cert: "certs/controller.crt"
    key: "certs/controller.key"
    ca: "certs/ca.crt"

8) Create a VM

VMs are created declaratively via YAML manifests:

apiVersion: kcore.io/v1
kind: VM
metadata:
  name: debian12-test
spec:
  cpu: 1
  memory: 2G
  image: https://cloud.debian.org/images/cloud/bookworm/latest/debian-12-generic-amd64.qcow2
  enableKcoreLogin: true
  nics:
    - network: default
      model: virtio

Apply it:

kctl apply -f vm.yaml
kctl get vms
kctl describe vm debian12-test

9) Manage VMs

kctl describe vm debian12-test
kctl delete vm debian12-test

Using Terraform

You can also manage VMs with the kcore Terraform provider:

terraform {
  required_providers {
    kcore = {
      source = "kcore/kcore"
    }
  }
}

provider "kcore" {
  controller_address = "localhost:9090"
  insecure           = true
}

resource "kcore_vm" "example" {
  name         = "debian12-test"
  cpu          = 1
  memory_bytes = 2147483648  # 2GB

  image_uri          = "https://cloud.debian.org/images/cloud/bookworm/latest/debian-12-generic-amd64.qcow2"
  enable_kcore_login = true

  nic {
    network = "default"
    model   = "virtio"
  }
}
