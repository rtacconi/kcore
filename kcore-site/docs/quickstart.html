Quick Start

Build the kcore ISO, boot a node from USB, and create your first VM.

kcore is alpha software. This guide assumes you have a dedicated machine or lab environment for testing.

Prerequisites

  • Nix with flakes enabled (for building the ISO)
  • Go 1.22+
  • Make
  • A USB drive (for booting the node)
  • A dedicated x86_64 machine or server for the kcore node

1) Clone and Build

git clone https://github.com/rtacconi/kcore.git
cd kcore

# Build the ISO
make build-iso

# Write to USB
USB_DEVICE=/dev/sdX make write-usb

2) Boot and Install

Boot the target machine from the USB drive, then log in with the default credentials (root / kcore).

# Run the installer — it writes to disk and configures everything
install-to-disk

# After reboot, all services start automatically
# The node agent self-registers with the controller

3) Configure the Controller

The controller runs on your workstation and manages the cluster. Create a config file:

# controller.yaml
databasePath: ./kcore.db
listenAddr: ":9090"

tls:
  caFile: ./certs/ca.crt
  certFile: ./certs/controller.crt
  keyFile: ./certs/controller.key

nodeNetworks:
  default: br0

Start the controller:

make controller
./kcore-controller --config controller.yaml

4) Configure kctl

kctl is the CLI for interacting with your kcore cluster. Create a config file at ~/.kcore/config:

# ~/.kcore/config
current-context: local

contexts:
  local:
    controller: "localhost:9090"
    insecure: true

For TLS-secured connections:

contexts:
  production:
    controller: "controller.example.com:9090"
    insecure: false
    cert: "certs/controller.crt"
    key: "certs/controller.key"
    ca: "certs/ca.crt"

5) Create a VM

VMs are created declaratively via YAML manifests. Create a file called vm.yaml:

apiVersion: kcore.io/v1
kind: VM
metadata:
  name: debian12-test
spec:
  cpu: 1
  memory: 2G
  image: https://cloud.debian.org/images/cloud/bookworm/latest/debian-12-generic-amd64.qcow2
  enableKcoreLogin: true
  nics:
    - network: default
      model: virtio

Apply it:

# Apply the manifest
kctl apply -f vm.yaml

# Check VM status
kctl get vms

# Get details for a specific VM
kctl get vm debian12-test

6) Manage VMs

# Describe a VM (detailed info)
kctl describe vm debian12-test

# Delete a VM
kctl delete vm debian12-test

Using Terraform

You can also manage VMs with the kcore Terraform provider:

terraform {
  required_providers {
    kcore = {
      source = "kcore/kcore"
    }
  }
}

provider "kcore" {
  controller_address = "localhost:9090"
  insecure           = true
}

resource "kcore_vm" "example" {
  name         = "debian12-test"
  cpu          = 1
  memory_bytes = 2147483648  # 2GB

  image_uri          = "https://cloud.debian.org/images/cloud/bookworm/latest/debian-12-generic-amd64.qcow2"
  enable_kcore_login = true

  nic {
    network = "default"
    model   = "virtio"
  }
}
