Architecture

kcore is composed of a control plane, node agents, and command-line/API clients.

Core Components

  • Controller: Stores desired state, schedules work, and serves the API.
  • Node Agent: Runs on each host, executes workloads through local virtualization APIs.
  • CLI/API Clients: `kctl`, Terraform, and automation interact with the controller.

High-Level Flow

kctl / Terraform / AI Agent | v Controller API + scheduler | v Node Agent (one per node) | v libvirt + storage + network

Reliability Model

  • Desired state is persisted and reconciled by the controller.
  • Agents report runtime status back to the control plane.
  • Failed operations can be retried without drift from source config.
