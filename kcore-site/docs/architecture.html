Architecture

kcore is a clustered KVM virtualization platform built with Go, gRPC, and NixOS.

Core Components

  • Controller — Written in Go. Stores desired state in SQLite, serves a gRPC API on port 9090, and coordinates VM operations across nodes.
  • Node Agent — Runs on each NixOS compute node. Self-registers with the controller on startup and manages local VMs through libvirtd.
  • kctl — CLI client that sends gRPC requests to the controller. Supports declarative manifests (kctl apply -f vm.yaml) and imperative queries (kctl get vms).
  • Terraform Provider — Manages VMs as Terraform resources via the same gRPC API.

Communication

All communication between components uses gRPC with Protocol Buffers. In production, connections are secured with mutual TLS (mTLS) using x509 certificates.

  • Clients (kctl, Terraform) → Controller: port 9090
  • Controller → Node Agents: port 9091
  • Node agents self-register with the controller and report status periodically

Host OS

Each compute node runs NixOS, providing:

  • Immutable system configuration — the entire OS is declaratively defined and reproducible
  • Atomic updates — system updates are all-or-nothing with instant rollback
  • Minimal attack surface — only the packages needed for virtualization are included in the ISO

Storage

Currently supports local storage and LVM. Cloud images are downloaded automatically when specified via image (YAML) or image_uri (Terraform). Storage backends are pluggable — ZFS and networked storage are on the roadmap.

Reliability

  • Desired state is persisted in the controller's SQLite database
  • Node agents report runtime status back to the controller
  • Failed operations can be retried without configuration drift
  • All services are managed by systemd with automatic restarts
