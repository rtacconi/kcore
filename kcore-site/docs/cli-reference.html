CLI Reference

kctl is the command-line interface for kcore. VMs are created declaratively via YAML manifests.

Global Flags

Flag Description
-c, --config Path to kctl config file (default: ~/.kcore/config)
-s, --controller Controller address (overrides config)
-k, --insecure Skip TLS certificate verification

Apply (Declarative VM Creation)

The primary way to create VMs. Apply a YAML manifest to the cluster.

# Create a VM from a manifest
kctl apply -f vm.yaml

# Dry run — preview what would be applied
kctl apply -f vm.yaml --dry-run

Example manifest (vm.yaml):

apiVersion: kcore.io/v1
kind: VM
metadata:
  name: debian12-test
spec:
  cpu: 1
  memory: 2G
  image: https://cloud.debian.org/images/cloud/bookworm/latest/debian-12-generic-amd64.qcow2
  enableKcoreLogin: true
  nics:
    - network: default
      model: virtio

Get (List Resources)

# List all VMs
kctl get vms

# Get a specific VM
kctl get vm debian12-test

# Wide output
kctl get vms -o wide

# JSON output
kctl get vm debian12-test -o json

# List all nodes
kctl get nodes

# List volumes
kctl get volumes

# List networks
kctl get networks

Describe

# Detailed VM information
kctl describe vm debian12-test

Delete

# Delete a VM
kctl delete vm debian12-test

# Force delete
kctl delete vm debian12-test --force

Create (Volumes and Networks)

The create command is used for volumes and networks, not VMs. Use kctl apply -f for VMs.

# Create a storage volume
kctl create volume my-volume --size 50G --storage-class local

# Create a network
kctl create network my-net

Pull

# Pull a cloud image
kctl pull https://cloud.debian.org/images/cloud/bookworm/latest/debian-12-generic-amd64.qcow2

Other Commands

Command Description
kctl version Print the kctl version
kctl controlplane status Check control plane status

Configuration

kctl reads its config from ~/.kcore/config by default. The config supports multiple contexts:

# ~/.kcore/config
current-context: local

contexts:
  local:
    controller: "localhost:9090"
    insecure: true

  production:
    controller: "controller.example.com:9090"
    insecure: false
    cert: "certs/controller.crt"
    key: "certs/controller.key"
    ca: "certs/ca.crt"

You can override the config with flags:

# Use a different controller
kctl get vms --controller node2.example.com:9091

# Force insecure even if config says otherwise
kctl get vms --insecure

Or with environment variables:

# Override config file location
export KCORE_CONFIG=/path/to/config
