Introduction to kcore

kcore is a next-generation virtualization platform that combines the reliability of traditional VM management with modern infrastructure-as-code practices and AI-powered automation.

Note: kcore requires a Linux host with KVM/libvirt support. See the installation guide for system requirements.

Installation

System Requirements

  • Linux host (x86_64)
  • KVM-enabled CPU (Intel VT-x or AMD-V)
  • 4GB+ RAM (8GB+ recommended)
  • 20GB+ free disk space
  • libvirt installed and running

Quick Install

Install kcore with a single command:

curl -sSL https://get.kcore.ai | sh

Or download and install manually:

# Download the latest release
wget https://github.com/kcore/kcore/releases/latest/download/kcore-linux-amd64.tar.gz

# Extract
tar -xzf kcore-linux-amd64.tar.gz

# Install binaries
sudo mv kcore-controller /usr/local/bin/
sudo mv kcore-node-agent /usr/local/bin/
sudo mv kctl /usr/local/bin/

# Verify installation
kctl version

Quick Start

1. Start the Controller

The controller manages the overall state of your infrastructure:

# Create a controller configuration
cat <<EOF > controller.yaml
listen_address: "0.0.0.0:50051"
database: "./kcore.db"
EOF

# Start the controller
kcore-controller --config controller.yaml

2. Configure kctl

Set up your CLI to connect to the controller:

# Create kctl config
cat <<EOF > ~/.kcore/config.yaml
controller_address: "localhost:50051"
insecure: true  # For local development
EOF

3. Create Your First VM

Using YAML definition:

# vm.yaml
apiVersion: v1
kind: VirtualMachine
metadata:
  name: my-first-vm
spec:
  memory: 2048
  vcpu: 2
  disks:
    - size: 20GB
kctl apply -f vm.yaml
Success! Your VM should now be running. Check its status with kctl get vms

Architecture Overview

kcore consists of three main components:

Controller

Central management server that maintains the desired state, handles scheduling, and provides the API interface for clients and the AI agent.

Node Agent

Runs on each compute node, manages local VMs using libvirt, reports status to the controller, and handles storage provisioning.

CLI (kctl)

Command-line interface for interacting with kcore. Create, manage, and monitor VMs and other resources with familiar kubectl-like commands.

Working with Virtual Machines

Creating VMs

# Quick create with CLI
kctl create vm web-server \
  --memory 4096 \
  --vcpu 4 \
  --disk size=50GB

# List all VMs
kctl get vms

# Get detailed info
kctl describe vm web-server

# Delete a VM
kctl delete vm web-server

VM Lifecycle

# Start a VM
kctl start vm web-server

# Stop a VM
kctl stop vm web-server

# Restart a VM
kctl restart vm web-server

Terraform Provider

kcore provides a native Terraform provider for infrastructure-as-code deployments:

terraform {
  required_providers {
    kcore = {
      source  = "kcore/kcore"
      version = "~> 1.0"
    }
  }
}

provider "kcore" {
  controller_address = "localhost:50051"
}

resource "kcore_vm" "app" {
  name   = "production-app"
  memory = 8192
  vcpu   = 4
  
  disk {
    size = "100GB"
  }
}

AI Integration with MCP

kcore includes a built-in Model Context Protocol (MCP) server that allows AI agents to interact with your infrastructure using natural language.

Example Agent Interaction

# Agent prompt
"Create a web application cluster with 3 VMs, 
 4GB RAM each, with a load balancer"

# Agent generates and executes:
resource "kcore_vm" "web_cluster" {
  count  = 3
  name   = "web-${count.index + 1}"
  memory = 4096
  vcpu   = 2
}
